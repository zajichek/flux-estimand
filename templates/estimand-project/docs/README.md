# Project Documentation

Use this directory for project-specific supporting documentation.

Examples include:

- rationale and background
- data sources
- parameter estimation
- execution plans
- validation reports
- results interpretation
- unresolved methodological decisions

The authoritative estimand remains the root [`ESTIMAND.md`](../ESTIMAND.md). Supporting docs may explain context, evidence, and implementation choices, but they should not redefine the decision, conditioning state, counterfactual actions or policies, outcomes, or contrasts.
